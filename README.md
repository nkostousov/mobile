# MOBILE APPLICATION TESTING

This repository contains test artifacts created during **functional and non-functional testing** of a mobile app — *Shopping List*.  
The main focus was verifying stability, usability, data saving, and behavior under interruptions or offline mode.

[Checklist Shopping_list Mobile App] (https://docs.google.com/spreadsheets/d/1AeGK9CEiA3x5Qj73MpNhIbnK1UFQWZkqlpz2LRoG-Y8/edit?usp=sharing) — detailed verification of installation, UI/UX, functionality, saving data, performance, and network scenarios.  

[Test_suites_Mobile_Testing_Shopping_List.pdf](https://github.com/user-attachments/files/23148012/Test_suites_Mobile_Testing_Shopping_List.pdf)  — structured test cases exported from Qase.io, including step-by-step actions and expected results.  

[Bug Reports Shopping List Mobile App](https://docs.google.com/spreadsheets/d/1k31Wbr-R_bGo-GW64DXM9hjhuQvyd_U6LZgEJrn_R-E/edit?usp=sharing) — list of defects found during functional and regression testing.

[Test_Run_Shopping_List_Mobile_App.pdf](https://github.com/user-attachments/files/23153212/Test_Run_Shopping_List_Mobile_App.pdf)  — final execution report exported from Qase.io with test results and statistics.

[Test Summary Report](https://docs.google.com/document/d/148ybOTPElchDDhJ9hKw_3J3iq7m_I6JuGUHnGzyWvic/edit?usp=sharing) — concise summary of testing results with key metrics, bug distribution, and project quality assessment.  

[WEB_Demo_Shopping_Charles_Proxy](https://drive.google.com/file/d/1cmfTzkw8WWdgDVdYWZyh_aH_uSahdQLB/view?usp=sharing) — video demonstration of network interception on the online store’s web version, using `Rewrite`, `Map Local`, `Map Remote`, and `Breakpoint`. 

[Mobile_Demo_Shopping_Charles_Proxy](https://drive.google.com/file/d/1wjODd8bqFZ6r_tPU0e_29CDqBlnFBwEY/view?usp=sharing) — video demonstration of proxy setup and traffic interception on a mobile device, including SSL proxying and request modification.

## What this repo demonstrates
- Experience in mobile app testing on Android (emulated device — Google Pixel 14).  
- Testing of installation, launch, UI/UX, offline mode, performance, and interruption handling.    
- Using **Charles Proxy** for network-level testing and debugging on both **web** and **mobile** platforms. 
- Understanding of mobile testing specifics (screen rotation, app state retention, data persistence).
